fhir-format

Accuracy-first Claude plugin for FHIR R4, R4B, and R5. Built to reduce confident-but-wrong guidance by enforcing claim labels, source-backed normative statements, and explicit boundaries between structural checks and full conformance.

R4 / R4B / R5 No guessed terminology codes Profile-aware answers Benchmark-backed
View Repository Read Docs See Benchmark

Quick install

claude plugin marketplace add https://github.com/dnlbox/fhir-format-skill
claude plugin install fhir-format@fhir-format-marketplace
/fhir-format:fhir-format

What it enforces

Claims are labeled as Normative, Version-delta, Profile-dependent, or Implementation practice. Normative claims should include authoritative references.

Best use cases

Required field clarifications, version differences, terminology-safe guidance, and conformance boundary explanations for modern TypeScript, Python, and AI workflows.

Independent validation

Includes reproducible relevance and CLI benchmarks so users can verify activation quality, answer quality, and source verifiability themselves.
