fhir-format

An accuracy-first Claude plugin for FHIR R4, R4B, and R5 that emphasizes source-aware guidance, explicit claim labels, and safer boundaries between local validation and full conformance workflows.

FHIR R4 / R4B / R5 Profile-aware reasoning No guessed terminology codes Benchmark-ready evidence
GitHub Repository Documentation Marketplace Draft

Quick install

claude plugin marketplace add https://github.com/dnlbox/fhir-format-skill
claude plugin install fhir-format@fhir-format-marketplace
/fhir-format:fhir-format

What it helps with

  • Required field questions with scoped context (base spec vs profile).
  • Version-aware interpretation across R4, R4B, and R5.
  • Terminology guidance with official lookup paths.
  • Clearer communication of validation boundaries.

Evidence and reproducibility

The repository includes two benchmark lanes: relevance routing and CLI grounding. Both are designed so external reviewers can rerun and inspect artifacts.
